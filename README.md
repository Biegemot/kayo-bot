[![Python Version](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![Telegram Bot API](https://img.shields.io/badge/Telegram%20Bot%20API-20.3+-blue.svg)](https://core.telegram.org/bots/api)

[![GitHub Release](https://img.shields.io/github/v/release/Biegemot/kayo-bot)](https://github.com/Biegemot/kayo-bot/releases)

Дружелюбный фурри-бот для Telegram с системой анкет, статистикой активности и ролевыми командами.

## ✨ Особенности

### 🤖 Основной функционал
- **Ролевые команды**: `/hug`, `/bite`, `/pat`, `/boop`, `/kiss`, `/slapass`
- **Система анкет** - заполнение и редактирование профилей фурсон через inline-режим
- **Статистика**: `/top`, `/today`, `/me` - рейтинги и личная статистика
- **Автоматические реакции** на сообщения с пушистыми ответами
- **Отдельные базы данных** для каждого чата (изоляция данных)

### 🖥️ Платформенная поддержка
- **Windows GUI** - графический интерфейс для удобного управления
- **Linux системная служба** - автоматическая установка и запуск
- **Docker контейнер** - кроссплатформенное развертывание
- **Консольный режим** - для всех платформ

### 🔄 Автоматизация
- **Автообновление** - проверка и установка обновлений каждые 6 часов
- **CI/CD пайплайн** - автоматическая сборка релизов
- **Конфигурация** - сохранение настроек при обновлениях

## 🚀 Быстрый старт

### 1. Клонирование и настройка
```bash
git clone https://github.com/Biegemot/kayo-bot.git
cd kayo-bot
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 3. Настройка токена
Создайте файл `.env` и добавьте ваш токен:
```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
```

### 4. Запуск
```bash
# Консольный режим
python main.py

# Windows GUI
python bot/gui.py

# Linux системная служба
./install_linux.sh
```

### Статистика и анкеты
| Команда | Описание |
|---------|----------|
| `/top` | Топ пользователей чата |

| `/me` | Моя анкета и статистика |

## 📋 Система анкет

### Поля анкеты
- 🐾 **Имя фурсоны** - имя вашей фурсоны
- 🦊 **Вид** - вид животного (лис, волк, кот, дракон и т.д.)
- 🎂 **Дата рождения** - дата рождения фурсоны
- 🔢 **Возраст** - возраст фурсоны
- 💕 **Ориентация** - ориентация фурсоны
- 🏙️ **Город** - город проживания
- 🤝 **Ищет** - кого ищет (друга, парту, девочку и т.д.)
- 🎭 **Тип личности** - выбор из предопределённых вариантов
- 🖼️ **Референс** - фото или ссылка на референс

### Редактирование анкеты
1. Отправьте `/me`
2. Нажмите кнопку "✏️ Редактировать"
3. Редактирование происходит в личном чате с ботом
4. Выберите поле для редактирования и отправьте новое значение

## 🏗️ Архитектура

### Структура проекта
```
kayo-bot/
├── main.py                 # Точка входа
├── bot/
│   ├── handlers/          # Обработчики команд
│   ├── services/          # Сервисы (БД, автообновление)
│   ├── gui.py            # GUI интерфейс (Windows)
│   └── data/             # Базы данных чатов
├── requirements.txt       # Зависимости Python
├── .env                  # Конфигурация (токен)
├── version.py            # Управление версиями
└── workflows/           # GitHub Actions
```

### Базы данных per-чат
- Каждый чат получает отдельный файл SQLite
- Данные изолированы между чатами
- Автоматическое создание при первом сообщении
- Хранятся в `bot/data/chat_<ID>.db`

## 📦 Сборка и релизы

### Автоматическая сборка
Проект использует GitHub Actions для автоматической сборки:
- **Windows .exe** - при обновлении версии
- **Linux binary** - при обновлении версии
- **Автоматический релиз** - создание тега и публикация

### Создание нового релиза
1. Обновите версию в `version.txt`
2. Запушьте изменения в ветку `main`
3. GitHub Actions автоматически соберёт и опубликует релиз

## 🛠️ Технические детали

### Зависимости
- `python-telegram-bot>=20.3` - API Telegram ботов
- `python-dotenv>=1.0.0` - управление переменными окружения
- `blessed>=1.20.0` - терминальные утилиты

### Версионирование
Используется семантическое версионирование:
- `MAJOR` - несовместимые изменения API
- `MINOR` - новая функциональность с обратной совместимостью
- `PATCH` - исправления ошибок

### Логирование
- Логи пишутся в `bot/logging/app.log`
- Уровни логирования: DEBUG, INFO, WARNING, ERROR, CRITICAL
- Ротация логов при достижении 10MB
