# Кайо — фурри-заяц Telegram бот

Дружелюбный пушистый кролик-бот для Telegram, который отслеживает активность и реагирует на сообщения с ролевыми командами и автоматическими реакциями.

## Особенности

- Ролевые команды: `/hug`, `/bite`, `/pat`, `/boop`
- Автоматические реакции на сообщения с пушистыми ответами
- Дашборд: `/top`, `/today`, `/me` для статистики и рейтингов
- Информационные команды: `/start`, `/help`, `/about`
- Отслеживание активности по чатам (отдельная база данных SQLite для каждого чата)
- Автообновление (проверка новых релизов при запуске и каждые 6 часов)
- Windows GUI приложение для удобного запуска и настройки
- Linux установщик для создания системной службы
- Сохранение конфигурации и данных при обновлении (не затрагивает `.env` и директорию `bot/data`)

## Установка

### Требования
- Python 3.8 или выше
- Telegram Bot Token (получите у @BotFather)

### Шаги установки

1. Клонируйте репозиторий:
   ```
   git clone https://github.com/Biegemot/kayo-bot.git
   cd kayo-bot
   ```

2. Установите необходимые Python-пакеты:
   ```
   pip install -r requirements.txt
   ```

3. Создайте файл `.env` на основе `config.example.env` и добавьте ваш токен Telegram бота:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен_здесь
   ```

### Альтернативный способ установки (Linux)

Для автоматической установки как системной службы:
```bash
chmod +x install_linux.sh
./install_linux.sh
```

Скрипт создаст виртуальное окружение, установит зависимости и настроит бота как systemd службу.

### Запуск в Docker

1. Соберите образ:
```bash
docker build -t kayo-bot .
```

2. Запустите контейнер:
```bash
docker run -d \
  --name kayo-bot \
  -e TELEGRAM_BOT_TOKEN=ваш_токен_здесь \
  -v $(pwd)/bot/data:/app/bot/data \
  kayo-bot
```

3. Просмотр логов:
```bash
docker logs -f kayo-bot
```

## Использование

Запустите бота одним из следующих способов:

### Обычный запуск (консоль)
   ```
   python main.py
   ```

### Windows GUI
    GUI приложение доступно через `bot/gui.py`. Для запуска:
    ```
    python -m bot.gui
    ```
    Или直接 запуск файла:
    ```
    python bot/gui.py
    ```
    GUI предоставляет интерфейс для запуска, остановки и настройки бота без использования консоли.

### Linux установщик (системная служба)
   ```
   ./install_linux.sh
   ```
   Скрипт создаст виртуальное окружение, установит зависимости, настроит `.env` (если отсутствует) и установит бота как системную службу через systemd.

Бот автоматически проверит наличие обновлений при запуске и каждые 6 часов. Если обнаружена более новая версия, он загрузит и установит её, не затрагивая файл `.env` и директорию `bot/data` (где хранятся per-чат базы данных).

## Команды

- `/start` - Запустить бота и получить приветственное сообщение
- `/help` - Показать справку с доступными командами
- `/about` - Информация о боте и версии
- `/hug [пользователь]` - Обнять пользователя (или себя, если пользователь не указан)
- `/bite [пользователь]` - Игриво укусить пользователя
- `/pat [пользователь]` - Погладить пользователя affectionately
- `/boop [пользователь]` - Ткнуть пользователя в нос
- `/top` - Посмотреть топ пользователей по общему количеству сообщений в текущем чате
- `/today` - Посмотреть топ пользователей по сообщениям за сегодня в текущем чате
- `/me` - Посмотреть свою статистику и титул в текущем чате

## Базы данных per-чат

Бот поддерживает отдельные базы данных SQLite для каждого чата, обеспечивая изоляцию данных. Каждый чат получает собственный файл базы данных, хранящийся в директории `bot/data/` (например, `data/chat_123456789.db`). Это гарантирует, что статистика и данные пользователей специфичны для каждого чата и не смешиваются между различными группами или личными чатами.

## Автообновление

При запуске бот проверяет последний релиз на GitHub (Biegemot/kayo-bot) и сравнивает его с текущей версией. Затем проверка повторяется каждые 6 часов. Если обнаружена более новая версия, он:
1. Скачает последнюю версию релиза
2. Заменит текущие файлы бота (исключая `.env` и директорию `bot/data`)
3. Перезапустит бота для применения обновления

Это гарантирует, что ваша конфигурация (токен бота и другие настройки в `.env`) и данные чатов останутся неизменными при обновлении.

## CI/CD

Проект использует GitHub Actions для автоматической сборки и релиза:

- **Сборка Windows exe**: При изменении `version.txt` в ветке `main`
- **Сборка Linux binary**: При изменении `version.txt` в ветке `main`
- **Автоматический релиз**: Создание тега и публикация релиза с артефактами

Для создания нового релиза:
1. Обновите версию в `version.txt`
2. Запушьте изменения в ветку `main`
3. GitHub Actions автоматически соберёт и опубликует релиз
