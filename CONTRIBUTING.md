Спасибо за интерес к проекту Kayo Bot! Мы рады любому вкладу и ценим ваше время и усилия.

## 🚀 Начало работы

### 1. Форк и клонирование
```bash
# Форкните репозиторий через GitHub
# Клонируйте свой форк
git clone https://github.com/YOUR_USERNAME/kayo-bot.git
cd kayo-bot
```

### 2. Настройка окружения
```bash
# Создайте виртуальное окружение
python -m venv .venv

# Активируйте окружение
# Linux/macOS:
source .venv/bin/activate
# Windows:
# .venv\Scripts\activate

# Установите зависимости
pip install -r requirements.txt
```

### 3. Создание ветки
```bash
git checkout -b feature/ваша-фича
# или
git checkout -b fix/исправление-ошибки
```

## 🏗️ Структура проекта

### Основные директории
```
kayo-bot/
├── main.py                 # Точка входа
├── bot/
│   ├── handlers/          # Обработчики команд Telegram
│   ├── services/          # Сервисы (БД, автообновление, логирование)
│   ├── gui.py            # GUI интерфейс для Windows
│   └── data/             # Базы данных чатов (не коммитить!)
├── requirements.txt       # Зависимости Python
├── .env.example          # Пример конфигурации
└── workflows/           # GitHub Actions
```

### Ключевые файлы
- `main.py` - Основная логика бота, регистрация команд
- `bot/handlers/` - Все обработчики команд (по одному файлу на группу команд)
- `bot/services/activity.py` - Управление активностью пользователей
- `bot/services/db_manager.py` - Управление базами данных
- `bot/services/auto_update.py` - Система автообновления
- `version.py` - Управление версиями

## 📝 Стиль кода

### Python код
- **Python 3.8+** - минимальная версия
- **PEP 8** - следуем стандартам
- **Type hints** - используем аннотации типов где возможно
- **Docstrings** - добавляем для всех функций и классов (Google style)

### Пример кода
```python
from telegram import Update
from telegram.ext import ContextTypes

def example_command(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    """
    Краткое описание команды.
    
    Args:
        update: Объект обновления Telegram
        context: Контекст выполнения
        
    Returns:
        None
    """
    try:
        # Логика команды
        user = update.effective_user
        update.message.reply_text(f"Привет, {user.first_name}!")
    except Exception as e:
        logger.error(f"Ошибка в example_command: {e}")
```

## 📊 Pull Request процесс

### 1. Подготовка изменений
```bash
# Добавьте файлы
git add .

# Создайте коммит с описательным сообщением
git commit -m "feat: добавлена команда /example"
# или
git commit -m "fix: исправлена ошибка в ActivityManager"
```

### 2. Формат сообщений коммитов
- `feat:` - новая функциональность
- `fix:` - исправление ошибки
- `docs:` - изменения в документации
- `style:` - форматирование, отсутствие изменений в коде
- `refactor:` - рефакторинг кода
- `test:` - добавление тестов
- `chore:` - обновление сборки, зависимостей

### 3. Создание Pull Request
1. Запушьте ветку: `git push origin ваша-ветка`
2. Создайте PR на GitHub
3. Заполните шаблон PR:
   - **Что изменено** - краткое описание
   - **Почему** - обоснование изменений
   - **Как протестировано** - шаги тестирования
   - **Чеклист** - [ ] Все пункты выполнены

## 🔧 Работа с базами данных

### Структура БД
- Каждый чат имеет отдельный файл SQLite
- Файлы хранятся в `bot/data/chat_<ID>.db`
- **Важно**: не коммитить файлы БД!

### Миграции
Если меняется схема БД:
1. Создайте скрипт миграции в `bot/services/migrations/`
2. Добавьте проверку версии схемы
3. Протестируйте на существующих данных
