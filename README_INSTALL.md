## 🚀 ИНСТРУКЦИЯ ПО УСТАНОВКЕ

### ШАГ 1: УСТАНОВКА ЗАВИСИМОСТЕЙ

#### Вариант A: С использованием sudo (рекомендуется)
```bash
# Установить pip3
sudo apt-get update
sudo apt-get install -y python3-pip

# Установить зависимости
cd kayo-bot
pip3 install -r requirements.txt
```

#### Вариант B: Без sudo (пользовательская установка)
```bash
# Установить pip в пользовательскую директорию
curl -sSL https://bootstrap.pypa.io/get-pip.py -o get-pip.py
python3 get-pip.py --user
rm get-pip.py

# Добавить pip в PATH
export PATH="$HOME/.local/bin:$PATH"

# Установить зависимости
cd kayo-bot
pip3 install --user -r requirements.txt
```

#### Вариант C: Виртуальное окружение
```bash
cd kayo-bot

# Создать виртуальное окружение (требует python3-venv)
sudo apt-get install -y python3-venv
python3 -m venv venv

# Активировать окружение
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### ШАГ 2: НАСТРОЙКА ТОКЕНА БОТА

1. Получите токен у @BotFather в Telegram
2. Отредактируйте файл `.env`:
   ```bash
   nano kayo-bot/.env
   ```
3. Замените `test_token_for_diagnostics_only` на ваш реальный токен:
   ```
   TELEGRAM_BOT_TOKEN=ваш_реальный_токен_здесь
   ```

### ШАГ 3: ЗАПУСК БОТА

```bash
cd kayo-bot
python3 main.py
```

```
kayo-bot/
```

## 📈 ЛОГИРОВАНИЕ

Система создает 5 типов логов в папке `logs/`:

1. **Основные логи** - `kayo-bot_YYYY-MM-DD.log`
2. **Структурированные логи** - `kayo-bot_YYYY-MM-DD.json`
3. **Логи команд** - `commands_YYYY-MM-DD.log`
4. **Логи базы данных** - `database_YYYY-MM-DD.log`
5. **Логи ошибок** - `errors_YYYY-MM-DD.log`

## ⚠️ РЕШЕНИЕ ПРОБЛЕМ

### Проблема: "ModuleNotFoundError: No module named 'telegram'"
**Решение:** Установите зависимости как описано выше.

### Проблема: "No module named 'dotenv'"
**Решение:** Установите python-dotenv.

### Проблема: Бот не запускается
**Решение:** Проверьте:
1. Токен в `.env` правильный
2. Все зависимости установлены
3. Нет ошибок в логах `logs/errors_*.log`
